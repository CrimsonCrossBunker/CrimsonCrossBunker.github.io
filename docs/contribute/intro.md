# 参与贡献

:::info[路线档案]
**前置**：愿意完成一次可验证的小改动　·　**最终成果**：让你的贡献进入正确的仓库或协作平台
:::

贡献不等于先学会 C++。选择一条与你当前技能相符的路线，从可在一天内完成的小任务开始。

## 选择路线

| 我想做…… | 需要什么 | 从哪里开始 |
|---|---|---|
| 修 bug、功能或构建 | Git、对应技术栈、可编译环境 | [代码贡献](./code) |
| 添加物品、配方或玩法 | 文本编辑器、JSON 基础 | [MOD 教程](/docs/mod/intro) |
| 翻译或校对词条 | 中英文理解、术语意识 | [翻译贡献](./translation) |
| 绘制或归置像素图 | 像素画工具，或 Python 工具环境 | [贴图贡献](./tileset) |
| 改进本站教程 | Markdown、实际验证步骤 | [网站仓库](https://github.com/LYHGLYTX/CCB-SITE) |

## 一次好的首次贡献

- 目标单一，可以清楚说出“修改前”和“修改后”。
- 先确认归属仓库、许可和提交入口。
- 按对应教程完成本地或平台验证。
- 附上必要的截图、日志、测试命令或来源说明。
- 接受维护者对范围、风格与兼容性的反馈。

## 贡献进入哪里

代码和内置内容进入 CCB 游戏仓库；贴图素材主要进入 CCB_UNDEAD_PEOPLE 贴图仓库；在线翻译进入 CCB 的 Transifex 项目；教程进入本站仓库。不要因为它们都属于 CCB 社区就把文件提交到同一个地方。

:::tip[先交流再投入]
较大功能、新系统、大批量翻译和整套贴图开始前，先到[社区](/community)说明想法并确认维护者需求。
:::
