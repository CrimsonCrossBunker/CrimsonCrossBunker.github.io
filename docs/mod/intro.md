# MOD 制作教程

:::info[路线档案]
**适合**：想添加物品、配方、怪物、地图或规则的创作者　·　**前置**：会编辑文本文件　·　**最终成果**：一个能独立加载和发布的 MOD
:::

CCB 的大量内容由 JSON 定义。制作 MOD 通常不需要修改 C++ 或重新编译游戏：把文件放进正确目录，创建新世界启用它，就能看到结果。

## 这条路线会做什么

| 课程 | 成果 |
|---|---|
| [创建第一个 MOD](./first-mod) | 游戏能识别并启用你的 MOD |
| [添加内容](./content) | 新增一个物品和一条制作配方 |
| [测试与发布](./test-and-publish) | 完成加载检查、实际验证和发布包 |

## MOD 能做与不能做的事

JSON 可以定义物品、配方、怪物、职业、场景、地图生成、对话、任务、效果与大量规则。需要新引擎能力、底层界面或性能机制的功能，才可能需要 C++。

遇到字段问题时按以下顺序查找：

1. 在 `data/json/` 或 `data/mods/` 搜索一个行为相近的现有对象。
2. 阅读 CCB 仓库的 `doc/JSON/JSON_INFO.md` 及对应专题文档。
3. 查看 `doc/MODDING.md` 与 `doc/MOD_COMPATIBILITY.md`。
4. 带上最小 JSON 和完整报错到[社区](/community)提问。

:::warning[为 CCB 当前版本开发]
CCB 会同步上游数据结构。不要只依赖多年以前的第三方教程；以当前 CCB 仓库中的文档和现有 JSON 为准。
:::

从[创建第一个 MOD](./first-mod)开始。
